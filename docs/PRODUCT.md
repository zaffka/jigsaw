# Jigsaw — Постановка задачи

## Суть продукта

Онлайн-игра с разрезными картинками (пазлами) для детей с расстройством аутистического спектра (РАС). Игра мотивирует ребёнка собирать изображения, вознаграждая за успешную сборку видеороликом, словом с озвучкой или анимацией.

Бэкенд — Go-сервис (REST API). Фронтенд — Preact + Tailwind, раздаётся как статика через Traefik (отдельно от API). Доступ через браузер на планшете, телефоне и десктопе. В перспективе — нативное Android-приложение, утилизирующее тот же REST API.

## Роли

### Ребёнок (игрок)
- Видит свою коллекцию пазлов
- Собирает пазл (drag-and-drop, touch-friendly)
- Получает награду за сборку (видео, слово с озвучкой, анимация)
- Видит прогресс (собранные пазлы, звёздочки)

### Родитель
- Регистрируется и создаёт профиль ребёнка
- Загружает изображение → система автоматически создаёт пазл
- Настраивает параметры пазла (количество кусков, тип нарезки)
- Загружает или выбирает награду (видеоролик, слово)
- Может предложить удачный пазл в общий каталог
- Видит статистику прогресса ребёнка

### Администратор
- Модерирует пазлы, предложенные в общий каталог (одобряет/отклоняет)
- Создаёт и публикует качественный контент в общий каталог
- Управляет пользователями (блокировка, сброс пароля)
- Видит общую статистику платформы

## Пользовательские истории

### Регистрация и вход
- Родитель регистрируется по email + пароль
- Опционально вход через Google / Apple (OAuth2)
- Родитель создаёт один или несколько профилей детей
- Ребёнок входит в свой профиль через упрощённый UI (аватарка, без пароля — выбор из профилей внутри родительского аккаунта)

### Создание пазла
1. Родитель загружает изображение (JPEG/PNG, до 10 MB)
2. Система валидирует формат и размер
3. Изображение нормализуется (ресайз под целевые разрешения)
4. Нарезка на куски (pkg/slicer) с выбранными параметрами
5. Куски сохраняются в S3
6. Родитель назначает награду: загружает видео (до 30 сек), вводит слово (опционально с автоозвучкой через TTS)
7. Пазл появляется в коллекции ребёнка

### Общий каталог
1. Родитель отмечает свой пазл как «предложить в каталог»
2. Пазл получает статус `pending_review`
3. Администратор видит очередь на модерацию
4. Администратор одобряет → пазл появляется в общем каталоге для всех пользователей
5. Администратор отклоняет → родителю приходит уведомление с причиной
6. Администратор может самостоятельно создавать контент напрямую в каталог (без модерации)

### Игровой процесс
1. Ребёнок выбирает пазл (из личной коллекции или общего каталога)
2. Видит картинку-образец и разбросанные куски
3. Перетаскивает куски на место (touch / mouse)
4. При правильном размещении — кусок «залипает»
5. После сборки — воспроизводится награда
6. Результат сохраняется (время, количество попыток)

## Нефункциональные требования

### Мультиязычность
- Поддержка нескольких языков с первого релиза: русский, английский
- Язык определяется по настройкам браузера (`Accept-Language`), с возможностью переключения вручную
- Родитель может зафиксировать язык в настройках профиля
- Интерфейс ребёнка — минимум текста (иконки, аватарки), но звуковые подсказки и слова-награды должны соответствовать выбранному языку
- TTS-озвучка слов-наград генерируется на языке, выбранном родителем
- Тексты UI хранятся в JSON-файлах (`web/locales/{lang}.json`), загружаются на клиенте
- API отдаёт локализованные строки (ошибки валидации, уведомления) по заголовку `Accept-Language` или настройке пользователя
- Каталожные пазлы могут иметь заголовки на нескольких языках (поле `titles jsonb` вместо `title text`)
- Добавление нового языка — только новый JSON-файл + переводы каталога, без изменений кода

### Доступность
- Touch-first: основная платформа — планшет/телефон
- Крупные элементы интерфейса, понятная навигация без текста для ребёнка
- Минимум отвлекающих элементов
- Поддержка оффлайн-режима для загруженных пазлов (в перспективе)

### Производительность
- Обработка изображения: < 10 секунд для стандартного пазла
- Загрузка игры: < 3 секунд на 4G
- Бандл UI: < 100 KB gzip (Preact + Tailwind purge)

### Безопасность
- Детский контент: никакого user-generated content без модерации в общем каталоге
- Сессии с HttpOnly/Secure/SameSite cookie
- Загружаемые файлы: валидация MIME, ограничение размера, antivirus-проверка в перспективе

## Технический стек

| Слой | Технология |
|------|-----------|
| Бэкенд | Go (REST API) |
| Фронтенд | Preact + Tailwind CSS, статика за Traefik |
| Игровой рендер | Canvas 2D |
| БД | PostgreSQL 18 |
| Объектное хранилище | SeaweedFS (S3-совместимый) |
| Аутентификация | Встроенная (bcrypt + сессии), опц. OAuth2 |
| Транспорт API | REST (net/http) + WebSocket для реал-тайма |
| Фоновые задачи | Очередь на PostgreSQL (SKIP LOCKED) |
| Обратный прокси | Traefik |
| Контейнеризация | Docker Compose |
